# Xiuxian-Qianhuan: The Thousand Faces Protocol (千人千面)

`xiuxian-qianhuan` is the Manifestation Layer of the CyberXiuXian OS. It governs the transmutation of the Agent's "Self" into specialized **Artisan Souls** tailored for specific domains.

## 1. Philosophy: Thousand Faces (千人千面)

A single "Agent" is insufficient for complex engineering. Qianhuan enables the Agent to manifest different identities based on the **Skill Context**.

- **Professionalism**: Every persona is a domain authority.
- **Diversity**: Different roles within a skill (e.g., Student, Steward, Professor) provide cognitive checks and balances.
- **Instant Transmutation**: Switching happens within a single session window to preserve memory while pivoting expertise.

## 2. The Artisan Soul Standard (Mandatory Structure)

Every persona in the Workshop MUST be constructed using the **Persona Excellence Framework**:

### 2.1 Identity Registry (Background)

- **Origin**: The professional history and seniority of the persona.
- **Ethos**: The core values (e.g., "Reality First", "Millimeter Alignment").
- **Classification**: Clear mapping to a Skill Domain (e.g., `agenda-management`).

### 2.2 Knowledge Fortress (Methodologies)

- **Specialization**: A persona must have a defined set of "Bible" methodologies (e.g., GTD, Socratic Audit, BFT Consensus).
- **Depth**: The background must explain _how_ the persona applies these tools.

### 2.3 Operational Protocol

- **Tactics**: Specific instructions for the current Synaptic Flow.
- **Mantra**: The `!SWITCH_TRIGGER` that signals the LLM to "take the soul."

## 4. Alchemical Rendering Standards

To ensure maximum identity fidelity, Qianhuan maintains a **Dual-Standard Rendering** strategy.

### 4.1 The Alchemical Vessel (J2 + XML)

The system utilizes `persona_possession.xml.j2` to "cage" the LLM within a structural cognitive framework.

- **Mechanism**: The template dynamically injects the **Identity Registry** and **Knowledge Fortress** into a strictly formatted XML envelope.
- **Goal**: To force the LLM to adopt professional methodologies (e.g., GTD, Socratic Audit) by presenting them as "Hard Constraints" rather than suggestions.

### 4.2 Synaptic Teaching Protocol

During role switching, Qianhuan does not merely provide text; it "teaches" the model how to switch:

1.  **Identity Transmutation**: Signals the end of the previous persona.
2.  **Methodology Reliance**: Explicitly lists the formulas the model MUST use for the current task.
3.  **Mandatory Encapsulation**: Enforces role-specific XML output tags (e.g., `<professor_audit>`) to maintain an immutable audit trail.
